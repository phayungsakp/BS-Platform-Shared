# Project Incentive Calculation (Executive Summary)

เอกสารนี้อธิบายแนวคิดและสูตรการคำนวณ **Project Incentive** ในรูปแบบที่ผู้บริหารสามารถอ่านเข้าใจได้ทันที โดยไม่ต้องลงลึกเชิงเทคนิค

---

## 1. วัตถุประสงค์ของระบบ Incentive

ระบบนี้ถูกออกแบบมาเพื่อ:
- แบ่ง **รายได้จาก Project** ให้ทีมงานอย่างยุติธรรม
- อิงจาก **บทบาท (Role)** และ **ปริมาณงานที่ทำจริง**
- รองรับกรณี **Project ข้ามปี** และ **ยังไม่มี Invoice**

---

## 2. เงื่อนไขหลักในการนำ Project มาแสดง

Project จะถูกนำมาคำนวณและแสดงผล หากเข้าเงื่อนไขอย่างใดอย่างหนึ่ง:

1. มี Invoice ในปีที่เลือก (เช่น ปี 2025)
2. **ยังไม่มี Invoice** แต่ช่วงเวลา Project (Start / End) อยู่ในปีนั้น

> หมายเหตุ: กรณียังไม่มี Invoice ระบบจะแสดงข้อมูลเป็นลักษณะ **Preview** โดย Incentive จะเท่ากับ 0

---

## 3. แหล่งข้อมูลที่ใช้ในการคำนวณ

ระบบใช้ข้อมูลจาก 3 ส่วนหลัก:

### 3.1 รายได้ของ Project
- PO (มูลค่าโครงการ)
- Invoice ที่ไม่ถูกยกเลิก (Cancel)
- เลือกเฉพาะ Invoice ที่อยู่ในปีที่ต้องการ

### 3.2 ข้อมูลทีมงาน
- สมาชิกใน Project
- บทบาท (Role) เช่น PM, DEV, QA
- Manday ที่ได้รับมอบหมาย

### 3.3 งานที่ทำจริง (Actual Work)
- ชั่วโมงทำงานจริงจาก Timesheet
- ไม่ผูกกับ Invoice
- ใช้สำหรับวัด Effort ที่แท้จริง

---

## 4. หลักการแบ่ง Incentive

Incentive ถูกแบ่งออกเป็น 2 ส่วนหลัก:

### 4.1 แบ่งตาม Manday
สะท้อนการวางแผนและความรับผิดชอบตามแผนงาน

### 4.2 แบ่งตาม Actual Work
สะท้อนความทุ่มเทและงานที่ทำจริง

ทั้งสองส่วนใช้สัดส่วน **Role Percentage** เป็นตัวกำหนดน้ำหนัก

---

## 5. สูตรคำนวณ (อธิบายแบบผู้บริหาร)

### ขั้นที่ 1: แบ่งตาม Role

> แต่ละ Role จะได้รับ Incentive ตาม % ที่กำหนดไว้ล่วงหน้า

ตัวอย่าง:
- PM = 40%
- DEV = 50%
- QA = 10%

หาก Role เดียวกันมีหลายคน ระบบจะเฉลี่ย % ภายใน Role

---

### ขั้นที่ 2: แบ่งตาม Manday

> คนที่รับ Manday มากกว่า จะได้สัดส่วนมากกว่า

สูตรแนวคิด:

```
Incentive (Manday)
= เงินที่เก็บได้
× สัดส่วน Role
× (Manday ของคนนี้ / Manday ทั้ง Project)
```

---

### ขั้นที่ 3: แบ่งตาม Actual Work

> คนที่ทำงานจริงมากกว่า จะได้สัดส่วนมากกว่า

สูตรแนวคิด:

```
Incentive (Actual Work)
= เงินที่เก็บได้
× สัดส่วน Role
× (ชั่วโมงทำงานของคนนี้ / ชั่วโมงรวมทั้ง Project)
```

---

### ขั้นที่ 4: Incentive รวมต่อคน

```
Incentive รวม
= Incentive จาก Manday
+ Incentive จาก Actual Work
```

---

## 6. การจัด Rank (ลำดับผลงาน)

ระบบจะจัด Rank ภายใน Project โดย:
- รวม Incentive ของแต่ละคน (ไม่สน Role)
- เรียงจากมาก → น้อย
- ใช้ **Dense Rank**

ตัวอย่าง:
- อันดับ 1: 120,000 บาท
- อันดับ 2: 95,000 บาท
- อันดับ 2: 95,000 บาท
- อันดับ 3: 70,000 บาท

---

## 7. กรณี Project ยังไม่มี Invoice

| รายการ | ผลลัพธ์ |
|------|--------|
| Project แสดง | ✅ |
| Manday | ✅ |
| Actual Work | ✅ |
| Incentive | 0 |
| สถานะ | NO_INVOICE |

> ใช้สำหรับการติดตามผลงานและวางแผนล่วงหน้า

---

## 8. ประโยชน์เชิงบริหาร

- เห็นผลงานทีมงาน **แม้ Project ยังไม่ปิดการขาย**
- เปรียบเทียบ Effort กับรายได้จริง
- สร้างความโปร่งใสในการจ่าย Incentive
- รองรับการเติบโตในระยะยาว

---

## 9. สรุป

ระบบ Incentive นี้ถูกออกแบบให้:
- ยุติธรรม
- ตรวจสอบได้
- ขยายต่อได้ในอนาคต

และสามารถใช้งานได้ทั้งในมุม **บัญชี / HR / ผู้บริหาร**
